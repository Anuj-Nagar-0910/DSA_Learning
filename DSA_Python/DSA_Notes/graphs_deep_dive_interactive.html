What are nodes and edges?

Node (vertex)

A thing — a data point, a place, a person, an entity.

Drawn as a circle. Holds a value or label.

Edge

A connection between two nodes — drawn as a line between circles.

Can carry direction, weight, or just be a plain link.

BST — Binary Search Tree

Full form breakdown

Binary — each node has at most 2 children (left & right)

Search — organized for fast searching: left < node < right

Tree — hierarchical, acyclic structure with one root

The golden rule

left < node < right

For every node: everything in its left subtree is smaller, everything in its right subtree is larger.

Why BST is powerful — O(log n) search

Each comparison eliminates half the tree. With 1,000,000 nodes: only ~20 comparisons needed. That's the power of O(log n) vs O(n) linear search.

def search(node, target):
    if not node or node.val == target:
        return node          # found or not here
    if target < node.val:
        return search(node.left, target)   # go left
    return search(node.right, target)      # go right

How graphs are stored in Python

1. Adjacency list — most common

from collections import defaultdict

graph = defaultdict(list)
graph['A'].append('B')
graph['A'].append('C')
graph['B'].append('D')
graph['C'].append('D')

# Result:
# {'A': ['B', 'C'], 'B': ['D'], 'C': ['D'], 'D': []}

Space: O(V + E). Best for sparse graphs.

2. Weighted adjacency list

graph = defaultdict(list)
graph['A'].append(('B', 5))   # (neighbor, weight)
graph['A'].append(('C', 3))
graph['B'].append(('D', 8))

# Iterate over weighted neighbors:
for neighbor, weight in graph['A']:
    print(neighbor, weight)    # B 5, then C 3

3. Adjacency matrix

# 4 nodes: A=0, B=1, C=2, D=3
matrix = [
    [0, 1, 1, 0],  # A connects to B and C
    [0, 0, 0, 1],  # B connects to D
    [0, 0, 0, 1],  # C connects to D
    [0, 0, 0, 0],  # D has no outgoing edges
]

# Check if edge A→B exists:
if matrix[0][1] == 1: print("A→B exists")

Space: O(V²). O(1) edge lookup. Best for dense graphs.
